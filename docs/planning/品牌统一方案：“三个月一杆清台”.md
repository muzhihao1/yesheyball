# 品牌统一方案：“三个月一杆清台”

## 一、核心原则

**一个核心，一个声音**。既然产品的核心卖点是“三个月一杆清台”，那么从产品名称、页面标题到所有关键文案，都必须围绕这个核心进行统一，形成强大的品牌认知和用户记忆点。

## 二、产品命名升级

### 1. 废弃旧名称
- **旧名称**：“中八大师之路”
- **问题**：过于宽泛，缺乏记忆点，无法体现产品的核心价值。

### 2. 新名称建议

**首选方案：就叫“三个月一杆清台”**

- **理由**：
  - **直接有力**：直接传达产品的核心承诺和价值主张。
  - **极易记忆**：口语化，朗朗上口，用户听一遍就能记住。
  - **自带营销属性**：这个名字本身就是一个强大的广告语。

**备选方案**：
- **清台挑战**：较为通用，但不如“三个月”有时间限定感。
- **90天清台**：数字感强，但不如“三个月”口语化。

**最终建议**：采用“三个月一杆清台”作为产品的正式名称。

## 三、核心文案体系

- **主Slogan（核心卖点）**：三个月一杆清台
- **副Slogan（定位补充）**：你的私人AI台球教练

## 四、全站文案与视觉统一实施方案

### 1. 页面标题（Title）
- **原则**：所有页面的`<title>`标签都应以新产品名称开头。
- **示例**：
  - 首页/挑战页：`三个月一杆清台 | 每日挑战`
  - 技能库页：`三个月一杆清台 | 技能学习`
  - 练习场页：`三个月一杆清台 | 自由练习`
  - 个人档案页：`三个月一杆清台 | 我的档案`

### 2. 页面主视觉（H1）
- **原则**：所有页面的左上角Logo区域，将“中八大师之路”替换为新名称“三个月一杆清台”。
- **视觉建议**：可以设计一个简约的Logo，将数字“90”或汉字“清”与台球元素结合。

### 3. 关键页面文案

**登录/注册页**：
- 将“欢迎来到中八大师之路”修改为“欢迎来到**三个月一杆清台**，开始你的蜕变之旅”。

**首页（90天挑战页）**：
- 页面主标题直接使用“**三个月一杆清台**”。
- 副标题可以是“系统化训练 · 能力提升 · 成为更强的球手”。

**App加载/启动页（如果未来有）**：
- 显著位置展示Logo和Slogan：“三个月一杆清台”。

### 4. 通知与提醒

# 功能整合方案：打造以“90天挑战”为核心的统一体验

## 一、核心整合策略

**一个主线，三大支撑**。以“三个月一杆清台”挑战（即90天挑战）为产品的唯一主线任务，其他所有功能模块都围绕这条主线，提供理论学习、练习巩固和数据反馈的支撑作用，彻底解决功能重复和定位模糊的问题。

| 模块原名称 | 新定位 | 核心功能 | 与主线的关系 |
| :--- | :--- | :--- | :--- |
| 90天挑战 | **主线任务中心** | 引导用户完成每日挑战，展示核心进度 | **主线** |
| 训练计划 | **技能学习库** | 提供“傅家俊十大招”的完整教学内容，供用户随时查阅、学习理论知识 | **支撑**（理论学习） |
| 关卡地图 | **自由练习场** | 提供游戏化的练习关卡，用于巩固在主线中学到的技能 | **支撑**（练习巩固） |
| 个人档案 | **数据与成就中心** | 统一展示所有数据：能力评估、训练统计、成就徽章 | **支撑**（数据反馈） |

## 二、具体功能整合方案

### 1. 能力评估整合：统一数据，消除不一致

- **问题**：存在两套不同的能力评估体系。
- **解决方案**：
  1. **统一评估体系**：废弃“个人档案”中的旧能力评估体系，全站统一使用“90天挑战”的“**清台能力分**”体系。
  2. **统一评估维度**：统一为 **准度、杆法、走位、发力、策略** 五个维度。
  3. **统一数据来源**：“个人档案”页面的“能力分析”模块，**不再自己计算**，而是直接调用和展示“90天挑战”页面的能力数据和雷达图。两个页面的数据必须**完全同步**。

### 2. 训练内容整合：主线驱动，内容解耦

- **问题**：“十大招”内容在“训练计划”和“90天挑战”中重复出现。
- **解决方案**：
  1. **“90天挑战”作为内容消费的唯一入口**：用户的每日训练任务**只在**“90天挑战”页面中进行。这些任务的内容来源于“十大招”体系的拆解和重组。
  2. **“训练计划”改造为“技能库”**：
     - 将“训练计划”标签页改名为“**技能库**”或“**技术百科**”。
     - 此页面不再提供独立的训练入口，而是作为一个**只读的参考资料库**。
     - 用户可以在这里系统地查看“十大招”每一招的完整理论知识、技术要点和视频讲解（如果未来有），但不能在这里“开始训练”。
     - 在“90天挑战”的每日任务中，可以提供一个链接，引导用户跳转到“技能库”的对应章节进行理论学习。

### 3. 训练进度整合：明确主次，统一展示

- **问题**：训练进度在多个页面重复展示。
- **解决方案**：
  1. **“90天挑战”作为主进度**：产品的核心进度就是90天挑战的完成天数。
  2. **“个人档案”作为进度汇总页**：
     - 在“个人档案”页面的最顶部，显著位置展示“90天挑战”的核心进度（例如“已挑战 XX / 90 天”）。
     - 移除“个人档案”中与“关卡地图”重复的“成长路径”模块。
  3. **“关卡地图”改造为“练习场”**：
     - 将“关卡地图”标签页改名为“**练习场**”或“**自由闯关**”。
     - 它的定位是**非强制**的、游戏化的练习巩固模块，用户可以在完成每日主线任务后，来这里进行额外的挑战和娱乐。
     - 它的等级和进度与“90天挑战”的主线进度**完全解耦**，作为一条独立的、次要的成长线。

### 4. 数据统计整合：数据归档，统一入口

- **问题**：训练统计数据分散在多个页面。
- **解决方案**：
  1. **“个人档案”作为唯一数据中心**：将所有与用户历史数据相关的统计模块，**全部整合**到“个人档案”页面。
  2. **移除其他页面的统计模块**：
     - **移除**“90天挑战”页面底部的“训练统计”模块（已完成天数、累计时长等）。这些信息应该在“个人档案”中查看。
     - “90天挑战”页面只保留与“今日任务”和“90天日历”相关的核心信息。
  3. **重新组织“个人档案”页面**：对“个人档案”页面进行重新布局，使其更有条理，例如分为“核心数据”、“能力分析”、“成就墙”、“历史记录”等板块。

## 三、新用户流程示例

1. **首次进入**：用户打开应用，直接进入“**三个月一杆清台**”（原90天挑战）页面。
2. **开始挑战**：页面引导用户点击“开始今日训练”，完成第1天的任务（例如“握杆”）。
3. **理论学习**：在训练任务中，系统提示“想深入了解‘握杆’的理论吗？点击这里前往**技能库**查看”，用户可选择跳转到“技能库”学习理论知识。
4. **巩固练习**：完成今日主线任务后，系统提示“今天的任务已完成！要去**练习场**挑战几个关卡放松一下吗？”，用户可选择跳转到“练习场”进行游戏化练习。
5. **查看数据**：用户可以随时点击“**我的档案**”标签，查看自己最新的“清台能力分”、训练时长、获得的成就等所有历史数据。

## 四、验收标准

- [ ] “训练计划”页面已更名为“技能库”，并移除了所有“开始训练”的入口。
- [ ] “关卡地图”页面已更名为“练习场”。
- [ ] “90天挑战”页面已移除底部的“训练统计”模块。

- [ ] “个人档案”和“90天挑战”页面的能力雷达图数据完全一致。
- [ ] 新用户流程顺畅，引导清晰，没有功能和内容的重复感官能上的困惑。
