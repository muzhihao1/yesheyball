# 前端开发任务清单（P0 优先级）

**目标**：解决影响新用户留存和持续使用的核心问题，为产品长期增长打下坚实基础。

---

## 1. 新手引导流程开发

**目标**：让新用户在首次登录后，能够清晰地知道“从哪里开始”和“接下来要做什么”，降低流失率。

### 1.1 欢迎页开发

- **页面**：新用户首次登录后显示的欢迎页面（模态框或独立页面）
- **UI 设计**：
    - 顶部显示产品 Logo 和欢迎语：“**欢迎来到耶氏台球训练系统！**”
    - 中间部分用三段式文案，共鸣用户痛点，明确产品价值（参考报告中的文案建议）
    - 底部有一个主按钮：“**开始我的水平测试**”
- **交互逻辑**：
    - 点击“开始我的水平测试”按钮后，进入水平测试流程
- **技术实现**：
    - 使用本地存储（localStorage）或用户数据库字段来判断是否为首次登录

### 1.2 水平测试流程开发

- **页面**：水平测试页面（3-5个问题，每个问题一页）
- **UI 设计**：
    - 每个问题页面包含：问题标题、问题描述、2-3个选项按钮
    - 示例问题1：“**你是否能稳定地进袋口球？**”
        - 选项A：“**几乎不能**”
        - 选项B：“**偶尔可以**”
        - 选项C：“**大部分可以**”
- **交互逻辑**：
    - 用户选择一个选项后，自动进入下一个问题
    - 完成所有问题后，显示一个加载动画，文案为：“**正在为你生成专属训练计划...**”
- **技术实现**：
    - 根据用户的回答，计算一个初始能力分数
    - 将能力分数和测试结果发送到后端，用于生成个性化训练计划

### 1.3 注册完成页改造

- **页面**：用户完成注册或水平测试后的页面
- **UI 设计**：
    - 移除或弱化“立即开始训练”的单一按钮
    - 增加一个“**接下来 3 天的训练计划**”卡片模块
    - 每个卡片包含：训练日期（例如：“第一天”）、训练主题（例如：“站位与出杆”）、预计用时（例如：“15分钟”）
- **交互逻辑**：
    - 用户可以点击任意一天的训练卡片，直接进入当天的训练任务
- **技术实现**：
    - 从后端获取用户的个性化训练计划数据，并动态渲染卡片

## 2. 首页（已登录）改造

**目标**：增强用户的进度感和每日回访动力。

### 2.1 “3个月一杆清台”进度可视化

- **页面**：首页（已登录）
- **模块**：Hero 区，即“90天台球挑战”模块上方
- **UI 设计**：
    - 增加一个可视化的进度条，显示“**距离一杆清台还剩 [X] 天**”
    - 进度条下方可以增加几个里程碑标记（例如：“基础阶段”、“进阶阶段”、“实战阶段”）
- **交互逻辑**：
    - 进度条根据用户的训练天数动态更新
- **技术实现**：
    - 从用户数据中获取已训练天数，计算剩余天数和进度百分比

### 2.2 日常任务卡片

- **页面**：首页（已登录）
- **模块**：Hero 区下方，或作为独立的“今日任务”模块
- **UI 设计**：
    - 一个包含3个任务的卡片列表
    - 每个任务包含：任务描述（例如：“完成今日训练”）、完成状态（复选框）、奖励（例如：“+10 XP”）
- **交互逻辑**：
    - 用户完成任务后，复选框自动勾选
    - 完成所有任务后，弹出一个“**恭喜你，完成所有日常任务！**”的提示，并自动发放奖励
- **技术实现**：
    - 每日任务列表从后端获取，或在前端根据用户状态动态生成
    - 任务完成状态需要与后端同步

### 2.3 连胜/连练提醒

- **页面**：首页（已登录）
- **模块**：顶部导航栏，用户头像旁边
- **UI 设计**：
    - 显示一个火焰图标和当前的连续训练天数，例如：“**🔥 3**”
    - 如果用户今天还没训练，图标变为灰色，并显示提示：“**今天还没训练哦，继续保持连胜！**”
- **交互逻辑**：
    - 用户完成今日训练后，图标和数字自动更新
- **技术实现**：
    - 从用户数据中获取连续训练天数，并根据最后训练日期判断今天是否已训练

## 3. 增长与分享功能开发

**目标**：利用社交关系，实现产品的自然增长和用户回访。

### 3.1 分享训练成绩

- **页面**：训练完成后的提示页面
- **UI 设计**：
    - 在“太棒了！今日训练已完成”的提示下方，增加一个“**分享我的成绩**”按钮
- **交互逻辑**：
    - 点击按钮后，生成一张精美的“训练成绩卡片”（图片格式）
    - 卡片包含：用户头像、用户名、训练日期、训练时长、星级评价、能力提升等信息
    - 弹出系统分享菜单，用户可以选择分享到微信、微博等社交媒体
- **技术实现**：
    - 使用 Canvas 或前端截图库（如 html2canvas）来生成成绩卡片图片
    - 调用浏览器的 Web Share API 或引导用户长按保存图片进行分享

### 3.2 邀请好友一起练

- **页面**：“我的”页面
- **模块**：增加一个“**邀请好友**”的入口（例如：一个独立的卡片或设置项）
- **UI 设计**：
    - 点击入口后，弹出一个模态框，包含：邀请文案、邀请链接、复制链接按钮、分享按钮
    - 邀请文案示例：“**我正在用‘耶氏台球’进行90天清台挑战，快来和我一起练吧！**”
- **交互逻辑**：
    - 用户可以复制邀请链接，或直接分享到社交媒体
    - 好友通过邀请链接注册后，双方建立好友关系
- **技术实现**：
    - 为每个用户生成唯一的邀请码或邀请链接
    - 后端需要处理邀请关系和奖励发放逻辑
